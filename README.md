#  Store Experiments & Features

This repository contains all **features, fixes, and experiments** implemented for specific store.

It helps the team:

* Reuse features when needed
* Keep a backup of all changes

---

# What’s inside?

Each **folder = one feature or experiment of store **

Inside every folder, you will find:

* `script.js` → makes the feature work
* `style.css` → controls how the feature looks
* `images/` → screenshots after applying the feature
* `README.md` → simple explanation of the feature

---

# How to check any feature

1. Open any folder
2. Open `README.md`
3. Look at the screenshots in the `images` section

This will show you what the feature does.

---

# Types of features you may find

* UI improvements (design changes)
* A/B testing experiments
* Bug fixes
* Tracking scripts

---

# How to use or reuse a feature

### 1. Open the feature folder

### 2. Copy the code:

* Copy from `script.js`
* Copy from `style.css`

### 3. Add it to the store:

* Paste CSS into custom CSS section
* Paste JS into custom JS or GTM

### 4. Test it:

* Check the website on desktop and mobile

---

# ⚠️ Important notes

* Some features are made specifically for this store
* Some may not work on other stores without changes
* Always test before using

---

# 📁 Example structure

```
feature-name/
├── script.js
├── style.css
├── README.md
├── images/
│   ├── after.png
```

---

# Naming convention

Each feature folder should have a clear name inside main folder which is its name is `store name`:

* `sticky-header`
* `checkout-upsell`
* `cart-improvement`
* `experiment-free-shipping-bar`

---

# For team members (adding new features)

When adding a new feature:

1. Create a new folder with a clear name
2. Add:

   * `script.js`
   * `style.css`
   * `images/` (screenshots)
   * `README.md`
3. Keep explanation simple and clear
4. if this store not have folder yet create folder with store name
